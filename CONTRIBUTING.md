# Contributing

## Summary
Contributions should improve technical accuracy, operational clarity, and cross-linking across overload-control concepts.

## Contribution guidelines
- Keep definitions precise and falsifiable.
- Include tradeoffs and failure modes.
- Add or update `Related concepts` links whenever adding new pages.
- Update `schemas/knowledge.json` and `schemas/content-index.json` for net-new documents.
- Prefer production examples over toy metaphors.

## Pull request checklist
1. Content includes a top-level **Summary** line.
2. Adjacent concepts are explicitly distinguished.
3. Operational metrics are listed where relevant.
4. Internal links resolve.
5. JSON schemas are valid JSON.

## Style
- Use one H1 per page.
- Use concise H2/H3 sections.
- Avoid marketing language and unsupported claims.
